## 2. 背景与目标
1. 顶部改为自定义透明栏，展示“龙城预约系统”。
2. 移除原顶部状态标签栏，在首页同时展示四类活动：我参与的、接受报名、未开始、已结束。
3. 每组活动使用横向卡片流，提升浏览效率。
4. 保持排行榜/我的页主体不改，仅保留底部悬浮 Tab 的角色差异。

### 3.1 In Scope（本次包含）

## 6. 分组规则（强制口径）
### 6.1 全局排除规则
- 活动状态为“已取消”或“已删除”的活动，不在任何分组中展示。

### 6.2 我参与的
- 条件（同时满足）：
  - 当前用户已报名；
  - 活动未结束；
  - 活动未取消；
  - 活动未删除。
- 排序：按活动开始时间从近到远（越近越靠左）。

### 6.3 接受报名
- 条件（同时满足）：
  - 活动状态为“未开始”；
  - 当前时间未到报名结束时间；
  - 报名开关开启；
  - 活动未满员；
  - 当前用户尚未报名该活动。
- 排序：活动开始时间越早越靠左。

### 6.4 未开始
- 条件（同时满足）：
  - 活动状态为“未开始”；
  - 且不在“接受报名”分组中出现。
- 排序：活动开始时间越早越靠左。

### 6.5 已结束
- 条件：活动状态为“已结束”。
- 排序：按活动开始时间降序（最近结束优先）。

### 6.6 全局不重复规则（强制）
- 同一活动在四个分组中只允许出现一次。
- 分组优先级：我参与的 > 接受报名 > 未开始 > 已结束。
- 若活动同时命中多个分组，仅归入优先级最高分组。

---

## 7. 交互规范
### 7.1 横向卡片流
- 每组独立横向滚动。
- 根据滑动距离吸附分页，每次仅切换 1 或 2 张卡片。
- 右划回看时，左侧已有历史卡片需露出 10px。

### 7.2 卡片点击与编辑链路
- 点击活动卡片弹出“活动详情”弹窗。
- 活动详情内新增“编辑”按钮：
  - 仅管理员可见，普通用户不可见。
- 点击“编辑”后弹出“活动编辑”弹窗。
- 样式不做大改，仅在编辑弹窗新增“活动类型”输入项。

---

## 8. 视觉与卡片样式
### 8.1 顶部栏
- 自定义透明样式。
- 标题固定“龙城预约系统”。
- 右上角浅色龙 Logo。

### 8.2 活动类型输入
- 创建活动与编辑活动均支持活动类型输入：
  1. 羽毛球
  2. 桌游
  3. 其它

### 8.3 本期卡片样式（以 UI 稿为准）
- 羽毛球：橙色渐变卡片。
- 桌游：蓝色渐变卡片。
- 其它：MP4 动态卡片。

### 8.4 类型文案显示（我参与的卡片右上角）
- 羽毛球：显示 `Badminton`
- 桌游：显示 `Boardgame`
- 其它：不展示文案

### 8.5 卡片底部信息区
- 采用透明毛玻璃效果。
- 展示字段：活动名称 / 活动备注 / 活动位置（简略地址）/ 参与人数。

### 8.6 人数展示格式
- 有人数上限：`报名人数/活动上限人数`
- 无人报名且有上限：`0/活动上限人数`
- 无人数上限：仅展示报名人数

### 8.7 头像与 MP4 规则
- 固定底色+头像卡片：展示最近报名 3 人头像；少于 3 人时从下往上递减。
- MP4 卡片：仅在该横向列表最左侧首卡位置播放，离开即暂停。
